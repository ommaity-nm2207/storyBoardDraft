Background and Inspiration

I've been interested in Video Games for a really long time. As of lately, I've been extremely interested in interactive games because I've enjoyed the idea of the story shaping according to the decisions you make. I've been quite excited at the prospect of making my own small video game with some witty dialogue and allowing the user to try and explore all the different options presented to them.

For inspiration, I've taken some ideas from a 2 games.

    Emily is away
    Sara is Missing

Objectives

I would like to make an interactive short story(more like a short teaser). There will be a few alternative pathways for the story to progress (currently looking at 2). This will be the core and primary objective.

If time permits, I would love to implement a feature which would let the user undo their actions so that they can explore more easily. A timer could also be added to the puzzle

Data

The dialogue which the user encounters will change depending on the input. The game will keep track of the sequence of inputs to change dialogue appropriately.

The maze will require a use of a 2d array to create a list of possible moves. Functions will be added to keep track of the users progress through the maze, as well as determine any conditions for game over (e.g hit the wall)

Since the game will require about 5 minutes per run, it might get frustrating to start the game from the beginning every time the page reloads. If time persists, I would like to implement a feature where the user can continue from where they left off (need to consult TA for what to learn for this feature)

Concepts I intend to use

Session 04 - Functions My game will determine the response using functions. There are multiple interactive elements. The buttons, tabs, textarea, passwords
Session 05 - Events

The game will use buttons to interact with the user. The game will listen for when the user clicks the button, and execute the appropriate response
Session 06 - My First Loop The game will go through some for loops to determine the response depending on the situation in the game
Session 08 - My First Dataset

My game won't be using a direct dataset, but it will be using arrays which contain responses and followup options. This was the easiest way I could think of to generalise the code, without hardcoding every possible response
Session 10 - My first Dashboard The dashboard will present some info and allow the user to switch through tabs at the top of the screen. The tabs will contain information regarding the game

Week 10

What is your project about?

Drifting apart

What is the data you plan to use?

Since I'm doing a game instead of a datastory, I don't have a direct data set. My game is a short interactive story. Since there are multiple options, I will make my own data set with all the responses and followup options

What is the question you plan to answer?

Will the 2 characters remain close friends?

Why is this an important question?

I've personally drifted apart from many friends even though we've been close friends in the past

Research has shown that past the 7th grade,differences in compatability naturally lead to friendships dissolving

However, all is not lost. It just means that people need to take more initiative and put in effort to keep a friendship alive. There are things people should try to do, and shouldn't do

How do you intend to explore this question?

I've added a few different story paths so that the user can experience what it's like to see a friendship drifting apart. The different paths explore the different kinds of responses someone might have.

Week 11

Describe the interactions you intend to use

I intend to let the user text Jake, the friend. However, since I want them to text the correct phrase, after they select the dailogue option, when they try to text, the message box will auto populate with the message on every key stroke

I want the user to be able to click through the different contacts and read through their text log

I want the user to try texting the password to the "AI" system, iris. It will advance the game depending on if the password is correct or not

I want the user to be able to click on the profiles and access the profile pictures of all the 3 different contacts

Week 12

Write the story board of the data story

User is a person who has recently moved to college, but was close friends with Jake. Due to some issue, he has been locked out of his phone

They have to type in the password. The player obviously doesn't know the password, but this will encourage him to look through the messages to look for the password

As the player reads the messages, they realise that they've drifted apart from their friends

After finding the password, the player can try to message the friend, and try to either fix or worsen the relationship
